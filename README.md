# Monopoly Admin Panel

**Monopoly Admin Panel** — админ-панель для проекта **TCP Monopoly**. Приложение подключается к игровому серверу по TCP, получает состояние игры в JSON-формате и позволяет администратору наблюдать за ходом партии и отправлять управляющие команды.

Проект написан на **C# / Avalonia UI** и является частью клиент-серверной версии настольной игры «Монополия».

## Возможности

- Подключение к TCP-серверу по IP-адресу.
- Отображение статуса соединения с сервером.
- Отображение текущего состояния игры:
  - статус игры;
  - текущий игрок;
  - локальное время наблюдения админки;
  - количество ходов с момента подключения админки.
- Отображение игрового поля с клетками, владельцами и фишками игроков.
- Отображение кубиков и последнего броска.
- Таблица статистики игроков:
  - покупки;
  - бонусы;
  - штрафы;
  - количество клеток под владением.
- Список игроков онлайн:
  - никнейм;
  - баланс;
  - цвет игрока;
  - статус: в лобби, в игре, вышел.
- История игровых событий от сервера.
- Общий чат с игроками.
- Отправка сообщений в чат от имени администратора по `Enter`.
- Админские действия:
  - изменить баланс игрока;
  - выдать штраф;
  - выдать бонус;
  - кикнуть игрока;
  - отправить команду запуска события;
  - отправить команду паузы / продолжения игры.

## Архитектура проекта

Проект построен вокруг главного экрана админ-панели, отдельных UI-компонентов и сервисов для работы с сетью.

```text
MonopolyAdminPanel/
├── Assets/                  # SVG-ресурсы игрового поля и кубиков
├── Models/                  # Модели данных
├── Services/                # Сетевой слой, обработка сообщений, парсинг JSON
├── ViewModels/              # Базовые ViewModel Avalonia-проекта
├── Views/
│   ├── Controls/            # Переиспользуемые UI-компоненты
│   ├── Dialogs/             # Диалоговые окна админских действий
│   ├── AdminPanelView.axaml # Основной экран админ-панели
│   └── MainWindow.axaml     # Главное окно приложения
├── App.axaml
├── Program.cs
└── MonopolyAdminPanel.csproj
```

### Основные компоненты UI

- `AdminPanelView` — основной экран, который координирует работу панелей и сервисов.
- `BoardView` — игровое поле.
- `GameInfoPanel` — общая информация об игре.
- `GameStatePanel` — статус игры, текущий игрок и время.
- `PlayerStatsPanel` — таблица статистики игроков.
- `OnlinePlayersPanel` — список игроков и их статусов.
- `DicePanel` — отображение кубиков.
- `EventHistoryPanel` — история событий.
- `ChatPanel` — общий чат.
- `ConnectionPanel` — статус подключения к серверу.
- `AdminActionsPanel` — кнопки действий администратора.

### Сервисы

- `NetworkService` — TCP-клиент, подключение к серверу, отправка и получение сообщений.
- `NetworkMessageHandler` — маршрутизация входящих JSON-сообщений по типам.
- `GameStateParser` — чтение игроков из `game_state`.
- `AdminMessageFactory` — создание JSON-команд от администратора.

## Сетевой протокол

Все сообщения передаются в JSON-формате:

```json
{
  "type": "message_type",
  "senderId": 0,
  "payload": {}
}
```

### Служебные идентификаторы

```text
server senderId = 0
admin  senderId = 65535
host player id = 1
```

По договорённости проекта игрок с `id = 1` является хостом. Если хост кикнут, игра считается завершённой.

## Подключение администратора

После ввода IP-адреса админка подключается к серверу и отправляет:

```json
{
  "type": "connect_request",
  "senderId": 65535,
  "payload": {
    "role": "admin"
  }
}
```

Сервер должен ответить `connect_accept` или `connect_reject`.

## Поддерживаемые входящие сообщения

Админ-панель обрабатывает основные сообщения сервера:

```text
connect_accept
connect_reject
server_disconnect
players_list_lobby
game_started
game_state
game_event
chat_message
game_paused
game_resumed
```

Кубики обновляются из `game_state`, если сервер присылает поля:

```json
{
  "lastDiceFirst": 3,
  "lastDiceSecond": 5
}
```

## Админские команды

Админка отправляет команды серверу через сообщение типа `admin_action`.

### Изменить баланс

```json
{
  "type": "admin_action",
  "senderId": 65535,
  "payload": {
    "action": "set_balance",
    "playerId": 1,
    "balance": 12000,
    "reason": "Коррекция баланса"
  }
}
```

### Выдать штраф

```json
{
  "type": "admin_action",
  "senderId": 65535,
  "payload": {
    "action": "fine",
    "playerId": 1,
    "amount": 500,
    "reason": "Нарушение правила"
  }
}
```

### Выдать бонус

```json
{
  "type": "admin_action",
  "senderId": 65535,
  "payload": {
    "action": "bonus",
    "playerId": 1,
    "amount": 500,
    "reason": "Бонус от администратора"
  }
}
```

### Кикнуть игрока

```json
{
  "type": "admin_action",
  "senderId": 65535,
  "payload": {
    "action": "kick",
    "playerId": 1,
    "reason": "AFK"
  }
}
```

### Сообщение в общий чат

```json
{
  "type": "chat_message",
  "senderId": 65535,
  "payload": {
    "text": "Сообщение от администратора"
  }
}
```

## Локальная статистика админ-панели

Часть статистики считается локально внутри админки и действует только в рамках текущей сессии администратора:

- количество ходов;
- время наблюдения за игрой;
- количество штрафов;
- количество бонусов;
- количество запущенных событий;
- локальная история покупок.

Количество клеток под владением берётся из данных сервера. Если игрок проиграл и сервер прислал пустой список владений, значение клеток у игрока падает до `0`, но количество покупок остаётся как историческое значение.

## Как запустить

### Требования

- Установленный .NET SDK.
- Запущенный сервер TCP Monopoly.
- Доступ к серверу по локальной сети или IP-адресу.

## Сборка

Для сборки проекта:

```bash
dotnet build
```

### Запуск из терминала

```bash
dotnet restore
dotnet build
dotnet run
```

После запуска нужно ввести IP-адрес сервера и подключиться.

## Особенности проекта

- Админка не является сервером и не хранит полное состояние игры как источник истины.
- Основное состояние приходит от TCP-сервера через `game_state`.
- Логи игровых событий не создаются локально, а выводятся после получения сообщений от сервера.
- Аукцион в админ-панели не реализуется и не обрабатывается.
- `senderId = 65535` зарезервирован для администратора.

## Статус проекта

Проект используется как админ-панель для TCP Monopoly.

Реализованы основные функции наблюдения за игрой, отображения состояния сервера и отправки админских команд.
